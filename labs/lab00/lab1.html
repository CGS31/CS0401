Logging In

Make sure your computer and monitor are turned on.  You should see a colorful pattern with the time and date on the screen.

Type <CTRL><ALT><Delete> at the same time on the keyboard.  You should see a login screen.

Log in to the machine by typing your Pitt account (network id and password) in the appropriate fields and hitting <Enter>. You may see some messages and after a few moments you should see your desktop.

Experiment with the desktop by moving the mouse around and seeing what (if anything) appears on your screen.

Looking at your desktop, you should see a trapezoid on the left of the bottom menu bar.  If you move your mouse over this it will change color.  Click on this icon and a number of application icons should appear on the screen with the word "Start" on the top left.

Click on the "Command Prompt" icon.  This should create a window with the label "Commend Prompt" within your desktop.  You will be using this window to compile and run your Java programs.   

Click now on the "Firefox" icon at the bottom of your window (looks like a planet with a fox curled inside).  You should now see a Firefox web browser window.  Type in the URL for this lab into the address window and hit <Enter>.  The lab should come up and now you can follow the rest of the lab online if you wish.  For the remaining labs (from Lab 2 on) hard copies will not be given out so you will need to view these in the web browser.

Accessing your Pitt AFS Files

While it is possible to issue commands through icons and mouse clicks, it is sometimes more convenient to type commands into a command prompt. If you are used to using a windowing / touchscreen environment, this might seem awkward at first, but will quickly become automatic to you.

Click your mouse on the command prompt window or its icon at the bottom of the screen so that it becomes the active window.

The prompt should show you your current working directory, which is likely

\Users\<yourPittID>

This is a hierarchical file system. The path above says that you are in the <yourPittID> directory, which is in the Users directory of your machine.

This directory above is local to the machine that you are using (i.e. it is specific to each computer -- not shared between them), and will not be backed up, so you don't want to save your regular files here.  Rather, you want to save your files in your Pitt AFS file space.  We will now cover how to access the files in your Pitt AFS file space.

In your command prompt type

kinit

You will then be prompted for your Pitt Network password.  Type it in and hit <Enter>.

Next, type

aklog

If you typed in your correct password, nothing should show on the screen, but you should now have a token to access your Pitt AFS files.  Verify this access by typing

tokens 

You should see a message saying:

Tokens held by the Cache Manager:
User <yourPittID>@UNIV.PITT.EDU's tokens for afs@pitt.edu ...

Now you can access your Pitt AFS files through your lab machine.  In the next steps you will see how to access these files both through the command prompt and through the graphical interface of your machine.

At your command prompt, type

N:

This will take you to the network drive on your machine, which is accessing the AFS root directory.  Your prompt should now be N:\>

Now type

cd pitt.edu\home\<FL>\<SL>\<yourPittID>\

where <FL> is the first letter of your Pitt ID and <SL> is the second letter of your Pitt ID.

You should now be at your AFS home directory.

You can list your files and subdirectories with the dir command.  If you have not used this account before you might have no files.  However, you should have at least two subdirectories in your main directory: public and private.  The public directory is used to store files that you wish other people to be able to read, such as your Web page files (if you have one).  The private directory is used to store files that can only be accessed by you.  All files you create for this course should be within your private directory (or within a subdirectory of your private directory).

Change to your private directory by typing

   cd private

To go back up one directory level, type

   cd ..

Experiment with the cd command so that you are comfortable moving from one directory to another in your account.  Now make sure that you are in your private directory.  Then type the commands:

 mkdir cs401
 cd cs401

This will make a new subdirectory within your private directory that you can use for your cs401 labs and projects, and change your current directory to that new subdirectory.  If you wish, you can make additional subdirectories within your cs401 directory for different labs and projects.

The command copy <FILE1> <FILE2> copies one file to another. You can copy the first Java handout from my Web directory to your current directory with the following command (all one line): 

  copy n:\cs.pitt.edu\usr0\ramirez\public\html\cs401\handouts\ex1.java .

You can view the contents of a file with the command type <FILE>. Thus, type ex1.java will (very quickly) print the contents of ex1.java on the screen. If that was too fast to read, you can type more ex1.java. Hitting the space bar will display subsequent pages and Control-C will interrupt the command.

Editing Files

You will need to create Java code files. You can do this using any editor you choose (there are several available – many are free).  It is not practical to edit your files in the command prompt, so you will use a graphical editor for this purpose.  However, to do this you need to get to your Pitt AFS files through a graphical environment.

Click on the icon that looks like a folder on the bottom menu bar of your computer.  You should see a number folders.  At the bottom of the window you should see the words "Network locations" and an icon that looks like a disk drive.  Double click on that icon.  This will take you into the same N:  network drive that you accessed earlier in the command prompt.

Now you must follow the path down to your AFS files.  Do this by double clicking on "pitt.edu" in your window.  Then double click on "home".  Then double click on the first letter of your id.  Then double click on the second letter of your id.

Now you are at your home AFS directory.

Continue by double clicking on "private" and then "cs401".

You should now see the "ex1.java" program that you just downloaded.  To edit the file right click on it, then select "Edit with Notepad++".

Try editing the file in various ways as you experiment with the Notepad++ editor.

Compiling and Running a Java Program

As discussed in lecture, running a Java application is a two-step process.  Go back to your command prompt window and try the following::

1) Make sure you are still in the correct directory.  It should show as your prompt.  If you must change your directory refer back to earlier sections of this lab.

2) Compile the program into byte code via the javac command:

    javac ex1.java

3) Run the byte code via the java command

    java ex1

Now go back to your Notepad++ editor and change the program.  Add some code or perhaps introduce an error into the code.  Go back to your command prompt and compile it again to see the result.

Exercise

As an exercise, go back to your Notepad++ editor, then click on File and then New   

Type in the code below.  When you are finished, click on the "Save" icon (the picture of an old floppy disk) and choose the name "Lab1.java"

 

// CS 0401 Lab1

// Practice Java program with a dual purpose:

// 1) To familiarize you with editing, compiling and running Java programs

// 2) To familiarize you with Java syntax

public class Lab1

{

    public static void main(String [] args)

    {

        int total = 87, number = 10;

        double doubleAve;

        int intAve;

        doubleAve = ((double)total)/number; // floating point division

        intAve = total/number;              // integer division

        System.out.println("The double average is " + doubleAve);

        System.out.println("The integer average is " + intAve);

        int value1 = 3 + 4 * 5;         // default precedence

                  int value2 = (3 + 4) * 5;       // change precedence with parens

        System.out.println("Value1 is " + value1);

        System.out.println("Value2 is " + value2);

    }

}

 

Once you have typed in and saved the program above, compile and run it from the command prompt using the java compiler and interpreter programs.  If you get any compilation errors, it is likely due to a typing error.  The compiler will indicate the line number of the error -- find it and correct the error, then compile the program again.
 

Java Versions

As you may know the Java language has evolved and different versions are now available.  Our lab is newly updated and uses a recent version of Java.  To check which version of Java you are using, in the command prompt window type

java -version

Most likely, you will see something like

java version "1.8.0_51"

followed by some other information. For this course it is important that the version of Java you use is 1.8 or higher.  If you have Java installed on your computer or you use some other machine, check the version to make sure it is at least 1.8.

What Can Go Wrong?

What can go wrong? Plenty. Here are some common problems.

You cannot access javac or java.  The javac and java programs are located in specific directories in the files of your machine.  To access these, the PATH, or location of the files, is put into a system variable.  If this variable is not set your machine will not be able to find the programs and they will not work.  In this case you should ask your TA for help.

You have no available AFS disk space. If you have used your allocation of disk space you might not be able create any new files. You will need to delete some files before you can continue. Ask your TA to help you with this.  It is also possible to increase your disk space quote (to a limited extent).  If you want to do this, ask at any of the CSSD Campus Computing Labs.

You have some other problem.  Ask your TA for help!
 

Downloading the JDK

If you want to work at home using your own computer, you will need to install the JDK yourself.  You can download a current version from the Java web site (now at Oracle) at http://www.oracle.com/technetwork/java/index.html. Despite Java's well-earned reputation for portability, you should remember that there are often small differences between programming environments. If you do your work outside the CS401 lab, you should compile and test it in the lab before submitting it. Your work is expected to compile and run on those machines.
